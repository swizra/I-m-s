Security Practices

KEFE is operated with practical storefront security in mind, including controlled account access, protected order data, monitored administrative actions, and a clear route for reporting suspected vulnerabilities.

Report a concern

security@kefe.ch

Report review target

New security reports are acknowledged within 2 business days.

Account protection

  • Customer and admin areas are separated by role and purpose.
  • Authentication is required before protected account information is shown.
  • Operational access is limited to users who need it for store management.

Application and data safeguards

Security controls are applied across authentication, data access, and order workflows. This includes role-based boundaries, server-side validation, and operational checks that help reduce unauthorized changes or exposure.

Storefront monitoring

We may review logs, order activity, and technical behavior to support reliability, investigate unusual events, and keep the service operating safely.

Customer security tips

  • Use a strong, unique password for your customer account.
  • Do not share account credentials or reuse admin login details.
  • Review order confirmations promptly and report suspicious activity quickly.

Reporting vulnerabilities

If you believe you found a security issue, please share a clear description, reproduction steps, affected page names, and any screenshots that help explain the problem without exposing unnecessary customer data.

Responsible disclosure

Please avoid actions that could degrade service, access personal data, or disrupt other users while validating a security concern. We prefer coordinated disclosure so issues can be assessed and resolved responsibly.
